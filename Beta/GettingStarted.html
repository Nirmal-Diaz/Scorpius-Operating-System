Name

Unlike most projects, "Project JavaScript Web Operating System" has the worst name ever. Because it represents only a third of its core technologies, The JavaScript web scripting language. With others being CSS(Cascading Style Sheets) styling language, HTML(Hypertext Markup Language), and PHP(PHP Hypertext Preprocessor) server-side scripting language.

Mission

At first, this project wasn't meant for sharing. It's soul purpose was to learn the core technologies of the web just by myself (Yeah, somewhat selfish). For that I planned to use the web for something what it's not meant to support (You guessed it, an operating system). It's like using waste material to build/invent something new (Except, the web is not waste material). So my mission was a success and I want to encourage other learners to try my method just for learning purposes. Who knows, it might work for you.

Scope

In JSOS, anything that is not a part of an inner <iframe>(i.e. An <iframe> element within the main window) belongs to the shell's scope.

File System Of The Shell

The "Shell" comes to life via the following file structure and its content.

                mainDirectory(Dir)
                    Start.html
                    FileSystem.php
                    System64(Dir)
                        Shell.css
                        Shell_Variables.css
                        UniversalFunctionsLibrary.js

HTML Hierarchy Of The Shell

                body
                    #taskArea
                        #windowStackArea
                            .windowStackAreaButton
                    #workspaceArea
                    #statusArea
                        #quickAccessArea
                    .window

Task Area (<div>)

Element.createdBy = (HTML Markup);

Element.id = "taskArea";

Element.CSSSelectors = (Class)"taskArea";

Element.events = null;

The "taskArea" is somewhat similar to the "Task Bar" in a Windows operating system. It holds the systemClock(not implemented), windowStackArea and other notifications(not implemented).

Window Stack Area (<div>)

Element.createdBy = (HTML Markup);

Element.id = "windowStackArea";

Element.CSSSelectors = (Class)"windowStackArea";

Element.events = (CSS Pseudo Class):hover;

The "windowStackArea" represents controls for minimizing/maximizing windows. The order of the windowStackAreaButtons directly matches with their windowZIndexArray index.

Window Stack Area Button (<span>)

Element-minimizeButton.createdBy = (JavaScript Function)createWindowStackArea;

Element-minimizeButton.id = windowIndex + "windowStackAreaButton";

Element-minimizeButton.CSSSelectors = (Child, Tag)"windowStackArea > span";

Element-minimizeButton.events = (CSS Pseudo Class):hover; (JavaScript)onclick:minimizeWindow;

Element-maximizeButton.createdBy = (JavaScript Function)createWindowStackArea;

Element-maximizeButton.id = windowIndex + "windowStackAreaButton";

Element-maximizeButton.CSSSelectors = (Class)"linkButton";

Element-maximizeButton.eventsList = (CSS Pseudo Class):hover; (JavaScript)onclick:maximizeWindow;

"windowStackAreaButtons" acts as controls for minimizing/maximizing windows.

Workspace Area

Element.createdBy = (HTML Markup);

Element.id = "workspaceArea";

Element.CSSSelectors = (Class)"workspaceArea";

Element.eventsList = null;

The "workspaceArea" is similar to the concept of a desktop. It contains blocks(not implemented), workspaceAreaIcons and the wallpaper. Created windows won't be appended to it as children, but will be appended as it's siblings.

Workspace Area Icons

Element.createdBy = (JavaScript Function)installApps;

Element.id = relevant appPath;

Element.CSSSelectors = (Class)"workspaceAreaIcon";

Element.eventsList = (CSS Pseudo Class):hover; (JavaScript)ondblclick:createWindow;

"workspaceAreaIcons" are automatically generated to represent ".html" files within the same directory as "Start.html".

Quick Access Area

Element.createdBy = (JavaScript Function)createQuickAccessArea;

Element.id = "quickAccessArea";

Element.CSSSelectors = (Class)"quickAccessArea";

Element.eventsList = null;

The origin of the "quickAccessArea" comes from the MacOS's "Menu Bar" that changes according to the currentlyFocusedWindow. It is the most prominent feature in JSOS. Most of the functions written into the "Shell" is for maintaining the "quickAccessArea". Content inside the quickAccessArea is decided by the developer of the application.

HTML Hierarchy Of The Quick Access Area

A developer doesn't need to develop a web application(i.e. a ".html" file) specifically for the JSOS. Because, most of the time JSOS is loosely-coupled with the application itself. But an application must contain the following HTML elements according to the given hierarchy in order to work with JSOS.

                #quickAccessArea
                    #dataCarrier (style="display:none;" data-window-index="" data-original-index="")

You can set the "style" to "display:none" of the quickAccessArea to prevent it being rendered in your application. And if you are to use the default quickAccessArea, it should have the following HTML hierarchy.

                #quickAccessArea
                    #dataCarrier (style="display:none;" data-window-index="" data-original-index="")
                    .quickAccessBar
                    #startButton
                    .quickAccessBar
